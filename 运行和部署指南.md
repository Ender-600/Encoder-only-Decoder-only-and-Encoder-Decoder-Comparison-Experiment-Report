## 🖥️ 本地运行

### 方法 1: Python 本地服务器（推荐✅）

```bash
# 1. 打开终端 (Terminal)

# 2. 进入项目文件夹
cd /Users/maxwell/Encoder-only-Decoder-only-and-Encoder-Decoder-Comparison-Experiment-Report

# 3. 启动服务器
python3 -m http.server 8000

# 4. 看到以下提示表示成功：
# Serving HTTP on :: port 8000 (http://[::]:8000/) ...
```

**然后在浏览器中打开：**
```
http://localhost:8000
```

**停止服务器：** 在终端按 `Ctrl + C`

## 🌐 在线部署

### 🟢 选项 A: GitHub Pages（免费，推荐）

#### 第一步：推送到 GitHub

```bash
# 1. 初始化 Git 仓库
cd /Users/maxwell/Encoder-only-Decoder-only-and-Encoder-Decoder-Comparison-Experiment-Report
git init

# 2. 添加所有文件
git add .

# 3. 提交更改
git commit -m "E³ Mini-Benchmark 项目页面初始版本"

# 4. 在 GitHub 创建新仓库
# 访问: https://github.com/new
# 仓库名: e3-benchmark（或您喜欢的名称）
# 设置为 Public（公开）
# 点击 "Create repository"

# 5. 连接并推送（替换 YOUR_USERNAME 为您的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/e3-benchmark.git
git branch -M main
git push -u origin main
```

#### 第二步：启用 GitHub Pages

1. 进入您的 GitHub 仓库页面
2. 点击顶部的 **"Settings"** (设置)
3. 在左侧菜单找到 **"Pages"**
4. 在 "Build and deployment" 部分：
   - **Source**: 选择 "Deploy from a branch"
   - **Branch**: 选择 `main` 和 `/ (root)`
   - 点击 **"Save"**
5. 等待 2-5 分钟

**您的网站将在以下地址可用：**
```
https://YOUR_USERNAME.github.io/e3-benchmark/
```

### 🟣 选项 B: Netlify（最快速，免费）

#### 方法 1: 拖拽部署（无需命令行！）

1. 访问 https://app.netlify.com/drop
2. 在 Finder 中打开项目文件夹
3. 将整个文件夹**拖拽**到浏览器页面中
4. ✅ **完成！** 立即获得 URL，例如：
   ```
   https://e3-benchmark-abc123.netlify.app
   ```

#### 方法 2: 连接 GitHub（自动更新）

1. 先完成上面的 GitHub 推送步骤
2. 访问 https://app.netlify.com
3. 注册/登录账号
4. 点击 **"Add new site"** → **"Import an existing project"**
5. 选择 **"GitHub"** 并授权
6. 选择您的 `e3-benchmark` 仓库
7. 点击 **"Deploy site"**

**优势**: 每次更新 GitHub 代码，网站自动更新！

## ❓ 常见问题

### Q1: 页面样式错乱怎么办？

**解决方案：**
```bash
# 1. 确保使用本地服务器运行（不要直接双击打开）
python3 -m http.server 8000

# 2. 清除浏览器缓存
# Mac: Cmd + Shift + R
# Windows: Ctrl + Shift + R
```

### Q2: 图片不显示怎么办？

**检查：**
1. 图片路径是否正确：`static/images/filename.png`
2. 图片文件是否存在于 `static/images/` 文件夹
3. 文件名大小写是否匹配（Mac 不区分，但 Linux 服务器区分）

### Q3: GitHub Pages 显示 404

**可能原因：**
1. 等待时间不够（需要 2-5 分钟）
2. Pages 设置不正确
   - 确保选择了 `main` 分支
   - 确保选择了 `/ (root)` 文件夹

**检查方法：**
```bash
# 在仓库 Settings → Pages 页面
# 查看是否有绿色提示：
# "Your site is live at https://..."
```

### Q4: 如何更新已部署的网站？

**GitHub Pages:**
```bash
# 修改文件后
git add .
git commit -m "更新内容"
git push

# 等待 1-2 分钟，网站自动更新
```

**Netlify (连接 GitHub 的情况):**
```bash
# 同上，推送到 GitHub 后自动更新
git push
```

**Netlify (拖拽部署的情况):**
- 重新拖拽整个文件夹到 Netlify Drop
- 或者在 Netlify 后台手动部署

### Q5: 本地运行提示端口被占用

**错误信息：**
```
OSError: [Errno 48] Address already in use
```

**解决方案：**
```bash
# 方法 1: 使用不同端口
python3 -m http.server 8001

# 方法 2: 找到并关闭占用 8000 端口的进程
lsof -ti:8000 | xargs kill -9

# 然后重新启动
python3 -m http.server 8000
```
